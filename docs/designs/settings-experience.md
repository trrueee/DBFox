# DBFox 设置体验设计

## 目标

设置页应表现为可信、克制的桌面数据工具，而不是通用后台表单。页面需要让用户快速理解当前配置、风险边界和下一步动作，并在复杂选项较多时保持渐进展开。

## 页面结构

1. 工作区标题负责说明页面目的，不在内容区重复标题。
2. 内容区使用最大 920px 的阅读宽度，并按领域拆分为独立 Section。
3. 每个 Section 包含标题、说明、可选图标和正文，不为每个字段额外套卡片。
4. 安全、网络和 AI 能力使用 Switch 控制，开启后在原位置展开配置。
5. 操作区固定在内容底部，左侧显示当前状态，右侧仅保留一个主操作。

## 视觉语言

- 品牌紫用于主操作、选中态和焦点环。
- 数据青用于数据、同步和来源语义。
- 成功绿、警告黄、危险红仅用于状态。
- 浅色和深色使用相同的表面层级：页面背景、分组表面、控件表面。
- 字体层级采用 12 / 13 / 14 / 18 / 24px，不使用小于 12px 的界面文字。
- 动效只表达开关、展开和异步状态，持续 150–250ms，并尊重 reduced motion。

## 共享组件

- `SettingsContent`：设置内容宽度和纵向节奏。
- `SettingsSection`：领域分组及说明。
- `SettingsField`：标签、控件、提示和字段错误。
- `SettingsToggle`：可访问的开关和渐进展开入口。
- `SettingsStatus`：测试、保存和错误状态。
- `SettingsActionBar`：状态与页面主操作。

## 页面约束

### 数据源

- 基础连接、安全与网络、数据能力分组。
- SSH 与 TLS 关闭时不展示其字段。
- 创建模式聚焦表单，不重复展示连接列表和“新建连接”动作。
- 测试结果在操作区内呈现，不插入大块警告卡片。

### 模型配置

- 模型服务与当前配置分组。
- API Key 提供显示/隐藏按钮和明确的安全存储说明。
- 测试连接与保存都展示进行中和完成状态。

### 诊断

- 运行环境与日志内容分组。
- “已脱敏”作为信任标记保留。
- 复制诊断包是主要操作，刷新和清空为次要操作；清空保持危险语义。
